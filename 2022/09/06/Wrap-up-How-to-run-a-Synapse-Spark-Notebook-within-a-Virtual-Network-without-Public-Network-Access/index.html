Wrap up: How to run a Synapse Spark Notebook within a Virtual Network without Public Network Access

パブリックネットワークアクセス無効で仮想ネットワークのみと通信可能な Synapse の Apache Spark ノートブックを実行する手順まとめ

パブリックネットワークアクセス無効で Azure Synapse Analytics を構築し、Apache Spark プールおよびノートブックを実行する方法について、ここまで 4回に渡って解説しました。
本記事では、それらのまとめを行います。

記事

1. Azure Syanpse Analytics を パブリックネットワークアクセス無効 で構築する

Construct Azure Synapse Analytics without Public Network Access

2. プライベートエンドポイント と プライベートリンクハブ を使って Azure Synapse Analytics を仮想ネットワークに接続する

Attach Virtual Network on Azure Synapse Analytics using Private Endpoint and Private Link Hubs

3. Azure Bastion 経由で接続する仮想マシンを構築する

Construct a Virtual Machine with Azure Bastion

4. 仮想ネットワークのみと通信が可能な Synapse Apache Spark プールでノートブックを実行する (Windows 仮想マシンと Bastion を利用)

Run a Notebook with Synapse Spark Pool within the Virtual Network via Synapse Studio using Windows Virtual Machine and Bastion

まとめ

パブリックネットワークアクセス無効にした Azure Synapse Analytics で Apache Spark プールおよびノートブックを実行する方法についてまとめました。
Azure の PaaS をプライベートネットワークで動作させるのは簡単ではないと思います。必要なことは大体ドキュメントに記載されていますが、実際に手を動かして構築するのが最も理解につながると思います。不要なリソースや常用しないリソースは削除か停止をすれば課金されないので、ぜひ手を動かして試してみてください。
